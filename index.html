Foreword

Foreword

The purpose of this presentation is to show how a reveal.js presentation can be customized using Tailwind CSS. The presentation was created from an Emacs org-mode file using org-reveal.

  • Press F1 to learn how to navigate the presentation. Also use the buttons on the bottom corners.
  • Since not all the customizations intended could be done using only Tailwind utilities, a custom CSS and JS files were also used.
  • A must for adapting this presentation to your needs is to check the Readme file of org-reveal.
  • For those not familiar with Reveal.js it is recommended to see the demo presentation to learn more about its capabilities.
  • Also check Tailwind documentation to learn how to install and more about its utilities since not all of them are covered here.

Thank you!

Tailwind utilities

Lists

Default list

  • In Emacs org-mode, a list item is defined with a dash symbol (-) at the beginning.
  • Subsequent items are defined in the same way.
  • Other symbols include: + and numbers (e.g. 1., 1)).
    • Numbers will result in an ordered list (<ol>) when exported to HTML
  • A list item can be moved up and down with the keyboard combination Meta + up or down
    • The Meta key refers to the Option or Alt keys on a Mac or Windows keyboards, respectively.

list-inside

  • Markers can be aligned to the left margin by using the utility: list-inside
    • In org-mode this is done including an html attribute (#+ATTR_HTML: :class list-inside) above the first item.

List vertically centered

  • This slide was produced with the snippet slide_1box_flex_col which sets the parent element to flex and flex-col
  • The attributes to the list were set to leading-relaxed
    • leading-relaxed will increase the space between items.
      • A custom value can be used such as leading-[2.5] but this has to be tested to avoid overflow.
        • From a brief test values <=3.3 would work for this amount of text/items.

Evenly spaced list

  • For a list (without nested items) it is possible to evenly space the items using a custom class flex-list.
  • The class should be equivalent to using the utilities flex and flex-col but these doesn't work.
  • This list was defined using: flex-list grow justify-evenly h-content.
  • Importantly grow will take all the vertical space defined with the custom class h-content.
